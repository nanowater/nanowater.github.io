이 글은 새 포스트를 작성할 때 가이드라인 역할을 하는 **템플릿 글**입니다.
`draft: true`로 설정되어 있어 배포(production) 상태에서는 노출되지 않고, 로컬 개발용 미리보기에서만 관찰할 수 있습니다.

새 글을 쓰실 때는 이 파일 전체를 복사해서 `src/content/blog/` 폴더 아래에 `새로운-파일명.md` 형식으로 파일을 만들어 내용만 고쳐 사용하시면 됩니다.

---

## 💡 Frontmatter 설정 속성 요약

1. **`title`**: 포스트 제목을 입력합니다. (제목의 해시값을 시드로 하여 오로라 그라디언트가 자동 설계됩니다.)
2. **`description`**: 포스트 요약 설명을 입력합니다. (SEO 메타 정보 및 SNS 카드 설명에 쓰입니다.)
3. **`pubDate`**: 포스트 최초 발행 일자입니다. (정렬 순서의 기준이 됩니다.)
4. **`updatedDate`**: (선택) 포스트 내용 수정 시, 상세 페이지에 수정일로 표시됩니다.
5. **`tags`**: (선택) 글과 관련된 태그 목록을 배열 형태로 기입하면 관련 태그 모아보기 페이지가 자동 빌드됩니다.
6. **`draft`**: `true`로 설정하면 배포 시 빌드 제외되어 비공개 처리됩니다.
7. **`heroImage`**: (선택) 오로라 자동 생성을 원치 않고 수동 이미지를 올릴 때 주석을 풀고 이미지 절대/상대 경로를 입력합니다.
8. **`showHero`**: 상단 대표 썸네일 노출 여부입니다. `false`로 주면 제목 영역만 나옵니다.
9. **`showComments`**: 하단 Giscus 댓글 기능 노출 여부입니다. `false`로 주면 댓글창이 가려집니다.
